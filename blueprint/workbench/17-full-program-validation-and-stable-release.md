# M17 — Full-Program Validation and Stable Release

**Goal:** prove that the complete eight-semester system attains its outcomes coherently, is teachable and sustainable, and can be released as a controlled bilingual curriculum.

**Indicative duration:** 4–6 months after first cohort evidence  
**Primary owners:** program board, assessment coordinator, external review chair, release manager  
**Depends on:** M08, M11, M14, M16  
**Exit unlocks:** stable operation and M18 annual cycle

## Epic E1 — Curriculum coherence audit

- [ ] **M17-E1-T01 [REV]** Validate every PLO progression, course outcome, prerequisite, program competence gate, project dependency, assessment, and resource ID; audit each course against §13.1 and every unit against §§13.12–13.13/workbench learner-instructor pilot gates, then resolve gaps/cycles/redundancy.
- [ ] **M17-E1-T02 [REV]** Reconcile actual workload/contact/lab time with credits across all semesters and identify overload/underload.
- [ ] **M17-E1-T03 [REV]** Audit mathematics/physics just-in-time coordination, repeated knowledge threads, tool transitions, and capstone readiness.
- [ ] **M17-E1-T04 [REV]** Audit EN/FR completeness, parity lag, assessment equity, terminology, accessibility, offline access, and learner cost.
- [ ] **M17-E1-T05 [REV]** Confirm local regulatory/accreditation mappings and clearly label any approval still pending.
- [ ] **M17-E1-T06 [REV]** Audit deployment capability: named local faculty/technician owners, computing/facilities, supplier/repair/manufacturing/service capacity, sample-bounded V3-readiness reproduce/service evidence, local/imported contribution, cost/access, and separation of the Benin profile from universal content.

## Epic E2 — Outcome and program-competence-gate evaluation

- [ ] **M17-E2-T01 [TEST]** Select at least one direct graduation measure per PLO plus enough earlier direct evidence—normally 3–4 measures total—to validate introduction/reinforcement; document artifact, individual status, sampling, rubric rows, decision rule, and limitations.
- [ ] **M17-E2-T02 [TEST]** Analyze G1–G9 attainment, reassessment, language, cohort, and progression evidence with appropriate caution.
- [ ] **M17-E2-T03 [TEST]** Audit M1–M8 portfolios from requirements through raw measurement, debugging, design, verification, context, service, and handover.
- [ ] **M17-E2-T04 [TEST]** Review graduate, placement, employer, partner, equipment, incident, access, workload, and project reliability evidence.
- [ ] **M17-E2-T05 [REV]** Identify observations, interpretations, actions, owners/dates, and future effectiveness checks.
- [ ] **M17-E2-T06 [DEC]** Confirm graduation per student: Level 3 in every PLO; Level 4 in PLO-07, PLO-08, PLO-11 and at least two depth-cluster technical PLOs; every G1–G9 passed; individual V2 design/verification evidence plus an attributable, defended contribution to a V3-readiness process. Cohort sampling moderates the system but never fills missing evidence.

## Epic E3 — Independent moderation

- [ ] **M17-E3-T01 [REV]** Appoint reviewers covering analog, digital/FPGA, embedded/software, PCB/manufacture, power/control, RF, curriculum, laboratory, bilingual EN/FR pedagogy/language equity, accessibility, international engineering expectations, and deployment-context professions.
- [ ] **M17-E3-T02 [REV]** Provide traceable evidence sample, not a curated showcase alone.
- [ ] **M17-E3-T03 [TEST]** Observe live teaching/lab/practical and independently reproduce selected builds/tests.
- [ ] **M17-E3-T04 [REV]** Record findings by critical/major/minor/advisory severity and publish disposition.
- [ ] **M17-E3-T05 [TEST]** Recheck critical and major closures.

## Epic E4 — Stable bilingual release

- [ ] **M17-E4-T01 [MAT]** Correct program-wide technical, dependency, workload, language, accessibility, safety, and operational defects.
- [ ] **M17-E4-T02 [TEST]** Rebuild all books, slides, code, simulations, HDL, PCB checks, indexes, offline bundles, and separately permissioned restricted packages from clean environments; physically and learner/instructor re-pilot every materially corrected unit before tagging.
- [ ] **M17-E4-T03 [SPEC]** Publish supported hardware/tool matrix, migration guides, known limitations, staffing/equipment requirements, local mappings, and deployment-mode boundaries.
- [ ] **M17-E4-T04 [REL]** Tag stable program release with checksums, change log, license/attribution, QA report, and archival copies.
- [ ] **M17-E4-T05 [DEC]** Approve next-cohort freeze and hand open improvements to M18.

## Required deliverables

- Complete curriculum/resource/DoD/coherence, workload, deployment-capability, and portability audits
- Direct PLO progression/graduation decision set, per-student G1–G9/level clearance, project/partner/operations evidence
- Independent review/reproduction report and severity-based disposition with rechecks
- Clean build plus targeted re-pilot evidence, stable EN/FR/offline/restricted releases, supported deployment profiles, migration/limitations/support plan

## Hands-on and real-life safeguard

The audit calculates not only lab hours but what learners actually did. Reviewers sample student agency, prediction, instrument use, construction/programming, fault diagnosis, model/measurement comparison, design choice, contextual requirements, and independent defense. A program can fail this milestone exit even with complete files if delivery remained passive.

## Exit criteria

- [ ] Every PLO has credible direct mastery evidence and no unmet G1–G9 program competence gate is hidden by averages.
- [ ] Every critical finding is closed; every major is closed or has a program-board-approved, time-bounded exception that does not violate an invariant. Otherwise this milestone does not pass and no stable release is tagged.
- [ ] Complete EN/FR source and offline outputs build reproducibly.
- [ ] Staffing, equipment, costs, approvals, known limits, and supported variants are explicit.
- [ ] Stable release and long-term owners are approved.
- [ ] The stable release demonstrates the exact graduation thresholds and local product-authority evidence; neither is inferred from polished showcases.
