# Instructor material

Instructor material is derived from the book, not a parallel explanation.

For each session, select a few chapter sections and specify:

- prerequisite retrieval and one prediction;
- the central model or representation;
- one worked example and its board/diagram cues;
- one demonstration, dataset, or fault diagnosis;
- questions that discriminate common misconceptions;
- a short transfer or exit check;
- optional cuts that do not break later dependencies.

Use [`lecture-template.qmd`](lecture-template.qmd). Slides may be generated from
that plan later, but they should remain visual teaching cues. Detailed prose,
references, and full derivations belong in the textbook.

Course schedules should reference chapter IDs and heading anchors so that a book
revision can be propagated without copying content.
