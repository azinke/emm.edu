# M03 laboratory qualification workspace

This directory implements the universal M03 controls required by the Course
Master Document. It deliberately separates reusable specifications from a named
deployment's approvals and measured evidence.

| Area | Repository evidence | Present boundary |
|---|---|---|
| [Safety](safety/README.md) | L0–L4 EN/FR system, energy-aware contract, induction, authorization/risk/inspection/drill/event records | institutional review, facility inspection, and live drills pending |
| [Platforms](platforms/README.md) | capability contracts, procurement/golden-test protocols, board-sheet/freeze/benchmark records | samples, physical tests, exact cohort freeze, and minimum-computer runs pending |
| [Procurement and benches](procurement/README.md) | quantity calculator, item/cost fields, receipt/maintenance, bench tests, design-pack and TCO controls | actual cohort, quotes, inventory, benches, packs, funding, and technician qualification pending |
| [Equipment paths](equipment-paths/README.md) | minimal/standard/advanced contracts, access rules, prepared data, EN/FR operations, requalification | physical comparison, fixture build, capacity pilot, safety/usability approval pending |

The dated deployment pack must name the institution, facility, jurisdiction,
cohort, staff, supplier channels, exact hardware/revisions, costs, and approvers.
Blank records, planning scenarios, and synthetic datasets are never promoted to
physical, financial, regulatory, or human-qualification evidence.
