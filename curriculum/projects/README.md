# Project spine

Projects integrate material that individual chapters deliberately separate. The
spine in [`spine.toml`](spine.toml) progresses from inspecting and measuring a
small system to qualifying and handing over a complete one.

A project begins with a need and measurable requirements. It includes interfaces,
risks, a configuration baseline, reviews, verification, individual evidence,
limitations, and a reproducible handover. A project is not complete merely
because a demonstration works once.
