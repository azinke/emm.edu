# Canonical record schemas

The JSON Schema 2020-12 contracts cover outcomes, prerequisite graphs, the
controlled termbase, platforms, components, projects, contexts, dated deployment
profiles and jurisdictions, local manufacturing/service capability, suppliers,
landed costs, facilities, V0–V4 contribution dossiers, claim/source registers,
safety, assessment, course and unit manifests, and the generated child backlog.

`validate_canonical.py` enforces the cross-record rules that a single document
schema cannot express: acyclic dependencies, known IDs, complete outcome maps,
paired-language currency, universal/deployment/context separation, dated claims,
and contribution/origin provenance. It is the required offline validation path;
third-party JSON Schema validators are an optional additional check.

Schemas are generated by `curriculum/scripts/generate_canonical_data.py`. Change
the generator and its seeded tests, not an individual generated schema.
